## Overview
Neonatal hearing screening relies on the detection of TEOAEs—faint acoustic echoes generated by the cochlea. In the DGS Titanius-12 scenario, the station's database was corrupted by non-stationary noise and nonlinear stimulus scrambling. 

This project implements a two-layer solution:
1. **Signal Processing:** A MATLAB-based pipeline to extract, align, and nonlinearly reconstruct the true OAE signal while filtering out extreme environmental noise.
2. **Decision & Matching:** A global optimization system that assigns a PASS/REFER classification based on split-half reproducibility (`RepeatCorr`) and maps the recovered signals to unlabelled templates using a Hungarian-style cost matrix.

*Included is a Python/Streamlit Graphical User Interface (GUI) to interactively explore the patient data, reconstructed waveforms, and spectral mappings.*

## Algorithm Pipeline

Our reconstruction and matching strategy follows a strict clinical DSP sequence:

1. **Jitter Correction:** Sub-sample alignment using stimulus onset cross-correlation (first 100 samples) to prevent destructive phase-cancellation.
2. **Nonlinear Summation:** Implementation of the $+1, +1, +1, -3$ stimulation paradigm to suppress linear stimulus artifacts.
3. **Biological Gating:** Application of a Gaussian time-gate centered at 8ms ($4\text{ms} < t < 16\text{ms}$) to isolate the cochlear response.
4. **Spectral Filtering:** A 10th-order IIR Bandpass filter ($1200\text{Hz} - 3800\text{Hz}$) coupled with active noise floor subtraction.
5. **Reproducibility Scoring:** Split-half correlation between odd and even epoch buffers to establish the PASS ($R \ge 0.54$) vs. REFER ($R \le 0.35$) threshold.
6. **Global Template Assignment:** Dual-domain scoring (time-domain morphology + frequency-domain mapping at JSON `centerFreqs`) fed into a competitive global cost matrix to resolve template similarity conflicts.
